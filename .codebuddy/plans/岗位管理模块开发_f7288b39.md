## 产品概述

新增一个岗位管理模块，用于管理系统中的岗位信息，支持岗位的增删改查操作，并建立用户与岗位的多对多关联关系。

## 核心功能

- 岗位管理：创建、更新、删除、查询岗位信息
- 岗位状态管理：启用/禁用岗位
- 用户岗位关联：一个用户可以拥有多个岗位，通过 `sys_user_post` 中间表建立关联
- 在用户模型中添加 `posts()` 关联方法，支持获取用户的所有岗位

## 技术栈

- 框架：PHP 8.x + Laravel ORM (Eloquent)
- 架构模式：MVC + Service + DAO 分层架构
- 数据库：MySQL

## 技术架构

遵循现有项目架构，按以下层次组织代码：

- **Controller 层**：处理 HTTP 请求，参数验证，调用 Service
- **Service 层**：业务逻辑处理
- **DAO 层**：数据访问封装
- **Model 层**：Eloquent 模型定义，关联关系

## 实现方案

参考现有的 `SysDept` 模块实现方式：

1. 创建 `SysPost` 模型，包含 `posts()` 关联方法
2. 创建 `SysUserPost` 中间表模型
3. 在 `SysUser` 模型中添加 `posts()` 多对多关联
4. 创建 `SysPostDao` 数据访问层
5. 创建 `SysPostService` 业务服务层
6. 创建 `PostController` 控制器

## 关键代码结构

### SysPost 模型关键字段

```
// fillable 字段
['post_code', 'post_name', 'post_sort', 'enabled', 'remark', 'created_by', 'updated_by']

// 状态常量
const ENABLED_DISABLED = 0;  // 禁用
const ENABLED_ENABLED = 1;   // 启用

// 关联关系
public function users(): BelongsToMany  // 岗位下的用户（多对多）
```

### SysUser 模型新增关联

```
public function posts(): BelongsToMany
{
    return $this->belongsToMany(
        SysPost::class,
        'sys_user_post',
        'user_id',
        'post_id'
    );
}
```

## 实现要点

1. 遵循现有代码风格：使用 `declare(strict_types=1)`、PHPDoc 注释、属性定义
2. 控制器使用 `#[Route]` 和 `#[Auth]` 属性定义路由和权限
3. 模型使用 `SoftDeletes` trait 实现软删除
4. DAO 层继承 `BaseDao`，实现 `setModel()` 方法
5. Service 层继承 `BaseService`，包含事务处理
