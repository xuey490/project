# 代码优化计划

根据 `SysRole` 和 `SysRoleService` 的代码风格和质量，对 `SysArticle`, `SysDept`, `SysMenu`, `SysPost`, `SysUser` 及其 Service 进行全面优化。

## 核心优化点

1.  **Controller 层标准化**：
    *   **参数获取**：统一使用 `$id = (int) ($request->query->get('id') ?? $request->request->get('id') ?? $request->attributes->get('id') ?? 0);` 模式获取 ID。
    *   **Body 解析**：统一使用 `$content = json_decode($request->getContent(), true) ?? $request->request->all();` 解析请求体。
    *   **响应格式**：统一使用 `JsonResponse` 返回 `['code' => 200, 'msg' => '...']` 或 `['code' => 200, 'data' => ...]`.
    *   **移除冗余**：移除 `SysArticle` 中多余的 `current_user` 判空（假设中间件已处理，若需传递则直接传）。

2.  **Service 层标准化**：
    *   **软删除 (Soft Delete)**：所有支持软删除的表（Dept, User, Role, Menu）在 `getList`, `getById` 中强制检查 `where('del_flag', '0')`。
        *   *注意*：`sys_post` 表目前没有 `del_flag`，将修改 SQL 文件添加该字段，并应用软删除逻辑。
        *   *注意*：`sys_article` 表目前没有 `del_flag`，将修改 SQL 文件添加该字段，并应用软删除逻辑。
    *   **事务 (Transaction)**：所有 `delete` 操作必须包裹在 `transaction` 中，并处理关联数据的清理（或检查）。
    *   **关联处理**：`create`/`update` 中统一使用 `sync` 处理多对多关系。

3.  **Schema 补全**：
    *   修改 `database/sql/init.sql` 和 `database/sql/demo_data.sql`，为 `sys_post` 和 `sys_article` 添加 `del_flag` 字段，默认值为 `'0'`。

## 执行步骤

1.  **Schema 更新**：修改 SQL 文件，为 `sys_post` 和 `sys_article` 添加 `del_flag` 字段。
2.  **SysPost 优化**：
    *   Service: 增加 `del_flag` 过滤，重写 `delete` 使用事务和软删。
    *   Controller: 统一参数获取和响应。
3.  **SysArticle 优化**：
    *   Service: 增加 `del_flag` 过滤，重写 `delete` 使用软删。
    *   Controller: 简化代码，移除冗余校验。
4.  **SysMenu, SysDept, SysUser 优化**：
    *   Service: 检查并统一事务使用和 `del_flag` 过滤。
    *   Controller: 统一参数获取和响应格式。
5.  **验证**：运行 `manual_admin_guard_and_crud.php` 确保功能未受损。
