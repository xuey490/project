# Implementation Plan: Complete Backend System with SQL

This plan details the steps to build the requested backend system, including the complete SQL schema for all modules.

## 1. Configuration

*   **Action**: Modify `config/database.php`.
*   **Detail**: Set `'engine' => 'laravelORM'` to enable Illuminate ORM.

## 2. Database Schema (SQL)

We will execute the following SQL to create the necessary tables.

### A. Department Table (`sys_dept`)
```sql
CREATE TABLE `sys_dept` (
  `dept_id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT 'Department ID',
  `parent_id` bigint(20) DEFAULT '0' COMMENT 'Parent Department ID',
  `ancestors` varchar(50) DEFAULT '' COMMENT 'Ancestors List',
  `dept_name` varchar(30) DEFAULT '' COMMENT 'Department Name',
  `order_num` int(4) DEFAULT '0' COMMENT 'Sort Order',
  `leader` varchar(20) DEFAULT NULL COMMENT 'Leader',
  `phone` varchar(11) DEFAULT NULL COMMENT 'Phone',
  `email` varchar(50) DEFAULT NULL COMMENT 'Email',
  `status` char(1) DEFAULT '0' COMMENT 'Status (0=Normal 1=Disable)',
  `del_flag` char(1) DEFAULT '0' COMMENT 'Delete Flag (0=Normal 2=Deleted)',
  `created_by` varchar(64) DEFAULT '' COMMENT 'Created By',
  `created_at` datetime DEFAULT NULL COMMENT 'Created Time',
  `updated_by` varchar(64) DEFAULT '' COMMENT 'Updated By',
  `updated_at` datetime DEFAULT NULL COMMENT 'Updated Time',
  PRIMARY KEY (`dept_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Department Table';
```

### B. User Table (`sys_user`)
```sql
CREATE TABLE `sys_user` (

  `sex` char(1) DEFAULT '0' COMMENT 'Sex (0=Male 1=Female 2=Unknown)',
  `avatar` varchar(100) DEFAULT '' COMMENT 'Avatar Path',
  `password` varchar(100) DEFAULT '' COMMENT 'Password',
  `status` char(1) DEFAULT '0' COMMENT 'Status (0=Normal 1=Disable)',
  `del_flag` char(1) DEFAULT '0' COMMENT 'Delete Flag (0=Normal 2=Deleted)',
  `login_ip` varchar(128) DEFAULT '' COMMENT 'Last Login IP',
  `login_date` datetime DEFAULT NULL COMMENT 'Last Login Date',
  `created_by` varchar(64) DEFAULT '' COMMENT 'Created By',
  `created_at` datetime DEFAULT NULL COMMENT 'Created Time',
  `updated_by` varchar(64) DEFAULT '' COMMENT 'Updated By',
  `updated_at` datetime DEFAULT NULL COMMENT 'Updated Time',

```

### G. Pivot Tables
```sql
-- User Role Pivot
CREATE TABLE `sys_user_role` (
  `user_id` bigint(20) NOT NULL COMMENT 'User ID',
  `role_id` bigint(20) NOT NULL COMMENT 'Role ID',
  PRIMARY KEY (`user_id`,`role_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='User-Role Pivot';

-- Role Menu Pivot
CREATE TABLE `sys_role_menu` (
  `role_id` bigint(20) NOT NULL COMMENT 'Role ID',
  `menu_id` bigint(20) NOT NULL COMMENT 'Menu ID',
  PRIMARY KEY (`role_id`,`menu_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Role-Menu Pivot';

-- Role Dept Pivot (For Custom Data Scope)
CREATE TABLE `sys_role_dept` (
  `role_id` bigint(20) NOT NULL COMMENT 'Role ID',
  `dept_id` bigint(20) NOT NULL COMMENT 'Dept ID',
  PRIMARY KEY (`role_id`,`dept_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Role-Dept Pivot';

-- User Post Pivot
CREATE TABLE `sys_user_post` (
  `user_id` bigint(20) NOT NULL COMMENT 'User ID',
  `post_id` bigint(20) NOT NULL COMMENT 'Post ID',
  PRIMARY KEY (`user_id`,`post_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='User-Post Pivot';
```

## 3. Foundation & Models

*   **BaseModel**: Create `App\Models\BaseModel` extending `Illuminate\Database\Eloquent\Model`.
*   **DataScopeTrait**: Create `App\Traits\DataScopeTrait` to implement the logic for the 6 data permission types.
    *   It will automatically filter queries based on the user's `data_scope` (1: All, 2: Custom, 3: Dept, 4: Dept+Sub, 5: Self).

## 4. Services & Logic

*   **AuthService**: Login logic, generating JWT tokens.
*   **SysUserService**: CRUD, assigning roles/posts.
*   **SysRoleService**: CRUD, assigning menus (permissions) and data scopes.
*   **SysMenuService**: Building the menu tree structure.
*   **SysDeptService**: Building the department tree structure.

## 5. Execution Order

1.  Switch ORM Config.
2.  Run SQL migration (Create `database/sql/init.sql` and run it or run via tool).
3.  Create Models & Traits.
4.  Create Services.
5.  Create Controllers (`Admin/SysUser`, `Admin/SysRole`, etc.).
6.  Verify functionality.
