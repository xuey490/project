# 角色分配用户与日志管理功能实现计划

## 1. 角色管理增强：用户分配
**目标**：在角色管理模块中，增加查询“已分配用户”和“未分配用户”的能力，并支持分配（授权）和取消分配（取消授权）。

**修改范围**：
*   **Service**: 修改 `App\Services\SysRoleService`
    *   新增 `allocatedUserList(int $roleId, array $params)`: 查询已分配该角色的用户列表。
    *   新增 `unallocatedUserList(int $roleId, array $params)`: 查询未分配该角色的用户列表。
    *   新增 `authUser(int $roleId, array $userIds)`: 批量给用户授权该角色。
    *   新增 `cancelAuthUser(int $roleId, array $userIds)`: 批量取消用户的该角色授权。
*   **Controller**: 修改 `App\Controllers\Admin\SysRole`
    *   新增 `allocatedUserList`: 对接 Service。
    *   新增 `unallocatedUserList`: 对接 Service。
    *   新增 `authUser`: 对接 Service。
    *   新增 `cancelAuthUser`: 对接 Service。
*   **Demo Data**: 在 `demo_data.sql` 中补充相关按钮权限（`sys:role:allocate` 等）。

## 2. 登录日志管理
**目标**：实现登录日志的查询与清理。

**修改范围**：
*   **Service**: 创建 `App\Services\SysLoginLogService`
    *   `getList(array $params)`: 支持用户名、状态、时间范围查询。
    *   `delete(array $ids)`: 批量删除。
    *   `clean()`: 清空日志。
*   **Controller**: 创建 `App\Controllers\Admin\SysLoginLog`
    *   `index`: 查询列表。
    *   `destroy`: 删除。
    *   `clean`: 清空。
*   **Demo Data**: 补充 `系统管理 -> 日志管理 -> 登录日志` 菜单及权限。

## 3. 访问日志管理
**目标**：实现访问日志的查询与清理。

**修改范围**：
*   **Service**: 创建 `App\Services\SysAccessLogService`
    *   `getList(array $params)`: 支持用户名、路径、状态码、时间范围查询。
    *   `delete(array $ids)`: 批量删除。
    *   `clean()`: 清空日志。
*   **Controller**: 创建 `App\Controllers\Admin\SysAccessLog`
    *   `index`: 查询列表。
    *   `destroy`: 删除。
    *   `clean`: 清空。
*   **Demo Data**: 补充 `系统管理 -> 日志管理 -> 访问日志` 菜单及权限。
