# PHP 后端与 Madong UI 前端全面对齐重构计划

## 目标
将现有的 PHP 后端完全重构，以匹配 `madong-ui-art-main` 前端项目的 API 接口定义、数据结构和权限模型。

## 1. 数据库架构重构 (Database Schema)
根据前端 `model.d.ts` 文件定义，重写 `init.sql` 和 `demo_data.sql`。

*   **表结构调整**：
    *   **通用变更**：所有主键统一为 `id`，状态字段统一为 `enabled` (0/1)，排序字段统一为 `sort`。
    *   **SysMenu**: `menu_name` -> `title`, `perms` -> `code`, `menu_type` -> `type` (1-6), `visible` -> `is_show`, 新增 `is_cache`, `is_affix`。
    *   **SysUser**: `phonenumber` -> `mobile_phone`, 新增 `real_name`。
    *   **SysRole**: `role_name` -> `name`, `role_key` -> `code`。
    *   **SysDept**: `dept_name` -> `name`。
    *   **SysPost**: `post_code` -> `code`, `post_name` -> `name`。
*   **新增表**：
    *   `sys_dict`: 字典主表 (`group_code`, `code`, `name`...)。
    *   `sys_dict_item`: 字典项表 (`label`, `value`, `color`...)。
    *   `sys_config`: 系统配置表 (存储 Site, Email, OSS 等配置)。

## 2. 核心框架调整
*   **响应格式 (Response)**：修改 `BaseJsonResponse`，将默认成功状态码从 `200` 改为 `0` (对应前端 `ApiStatus.success = 0`)。
*   **路由配置 (Routes)**：重写 `routes.php`，路由前缀从 `/Admin` 改为 `/system`，并严格匹配前端路径 (e.g., `/system/user`, `/system/auth/user-info`)。

## 3. 业务模块重构 (Controllers & Services)
*   **认证模块 (Auth)**:
    *   `POST /system/login`: 返回 Token。
    *   `GET /system/auth/user-info`: 返回用户信息 + 角色 + 权限。
    *   `GET /system/auth/user-menus`: 返回构建好的路由菜单树。
    *   `GET /system/auth/perm-code`: 返回权限码列表。
*   **系统管理 (System)**:
    *   **User**: 适配新字段，支持重置密码、分配角色等接口。
    *   **Role**: 适配新字段，支持菜单权限分配。
    *   **Menu**: 适配 `type` (1-6) 和新字段，返回树形结构。
    *   **Dept**: 适配新字段，返回树形结构。
    *   **Post**: 适配新字段。
    *   **Dict**: 实现字典及字典项的 CRUD。
    *   **Config**: 实现 `/system/config/info` (获取配置) 和 `/system/config` (保存配置)。
    *   **Logs**: 适配登录日志和访问日志接口。

## 4. 模型层更新 (Models)
*   更新所有 Model 类 (`SysUser`, `SysRole` 等) 以匹配新的表名和列名，确保 Eloquent 关联关系正确。

## 5. 验证
*   更新 `tests/manual_admin_guard_and_crud.php` 脚本，模拟前端请求，验证接口路径、参数接收和响应格式是否完全一致。

**执行顺序**：
1.  重写 SQL 文件 (init & demo)。
2.  更新 Models。
3.  更新 BaseJsonResponse 和 Routes。
4.  逐个实现/更新 Controllers 和 Services。
5.  验证。
