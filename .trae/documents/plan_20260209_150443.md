# 完善职位管理与部门多负责人支持

## 背景
用户要求“继续”上一轮工作。经检查，上一轮工作已完成 Admin 模块的核心鉴权（白名单、菜单 perms 拦截）与大部分基础数据的 CRUD。但遗漏了“职位管理（SysPost）”的控制器与服务实现，且“部门（SysDept）”目前仅支持单负责人（`leader` 字段）。

## 目标
1.  **补齐职位管理模块**：实现 `SysPostService` 与 `SysPost` 控制器，支持 CRUD 与 Status 变更。
2.  **完善部门负责人支持**：虽然不修改表结构（避免破坏现有数据兼容性），但将在 Service 层支持 `leader` 字段存储逗号分隔的多个负责人名称，并在文档或注释中说明。
3.  **验证**：通过测试脚本验证职位管理的增删改查。

## 计划步骤
1.  **创建 SysPostService**：实现 `getList`, `create`, `update`, `delete`, `changeStatus`, `getById`。
2.  **创建 SysPost 控制器**：暴露上述服务接口，保持统一的 `BaseJsonResponse` 风格。
3.  **更新 Demo 数据**：在 `demo_data.sql` 中加入职位管理的菜单与按钮权限（已在上一轮部分完成，需确认完整性）。
4.  **验证**：运行 `manual_admin_guard_and_crud.php`（上一轮已包含对 Post 的部分测试，本次将确保其完全通过）。

## 补充说明
-   部门多负责人：将在 `SysDeptService::create/update` 中允许 `leader` 字段传入逗号分隔字符串（如 "zhangsan,lisi"）。
-   数据权限：`SysPost` 暂时不需要数据权限过滤（通常职位是全局字典），除非用户后续明确要求按部门隔离职位。
