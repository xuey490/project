# 多租户功能实现总结

## 一、已完成的实现

### 1. TenantContext 优化（使用 Symfony Request）

**文件**: `/workspace/framework/Tenant/TenantContext.php`

**改进点**:
- ✅ 使用 Symfony Request 属性存储租户信息，避免静态变量污染
- ✅ 支持 RequestStack，兼容 Workerman 常驻进程环境
- ✅ 提供 `setTenantId()`、`getTenantId()`、`shouldApplyTenant()` 等方法
- ✅ 支持超管模式 `ignore()` / `restore()` / `withIgnore()`

**使用示例**:
```php
// 在中间件中设置
$requestStack = new RequestStack();
$requestStack->push($request);
TenantContext::setRequestStack($requestStack);
TenantContext::setTenantId(1001);

// 在模型中使用
if (TenantContext::shouldApplyTenant()) {
    $builder->where('tenant_id', TenantContext::getTenantId());
}
```

### 2. JWT Token 方案

**文件**: `/workspace/framework/Tenant/JwtTenantContext.php`

**功能**:
- ✅ 生成包含租户ID的 JWT Token
- ✅ 从 Token 解析租户ID、用户ID
- ✅ Token 验证和过期检查
- ✅ Token 刷新机制
- ✅ 从 Authorization Header 提取 Token

**使用示例**:
```php
// 生成 Token
$token = JwtTenantContext::generateToken([
    'user_id' => 1001,
    'username' => 'admin',
    'tenant_id' => 2001,
], $secret);

// 解析 Token
$tenantId = JwtTenantContext::getTenantIdFromToken($token, $secret);
$userId = JwtTenantContext::getUserIdFromToken($token, $secret);
```

### 3. Session/Cookie 方案

**文件**: `/workspace/framework/Tenant/SessionTenantContext.php`

**功能**:
- ✅ 将租户信息存储在 Symfony Session 中
- ✅ 支持租户切换
- ✅ 支持多个租户列表管理
- ✅ 与 TenantContext 同步

**使用示例**:
```php
// 登录时设置 Session
SessionTenantContext::setTenantSession($tenantId, $userId, $tenants);

// 切换租户
SessionTenantContext::switchTenant($newTenantId);

// 同步到 TenantContext
SessionTenantContext::syncToTenantContext();
```

## 二、使用指南

### 1. 数据库迁移

```bash
# 执行 SQL 迁移文件
mysql -u root -p your_database < database/migrations/2026_03_19_000001_create_tenant_tables.sql
```

### 2. 配置中间件

在您的路由或中间件配置中添加：

```php
use App\Middlewares\TenantMiddleware;

// 在路由中使用
$router->middleware(TenantMiddleware::class)->group(function ($router) {
    $router->get('/api/users', [UserController::class, 'index']);
});
```

### 3. 登录流程

```javascript
// 前端登录示例
const login = async (username, password, tenantId) => {
    const res = await fetch('/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password, tenant_id: tenantId })
    });
    const data = await res.json();
    
    // 保存 Token
    localStorage.setItem('token', data.data.token);
    localStorage.setItem('tenant_id', data.data.tenant_id);
};
```

### 4. API 请求

```javascript
// 发送请求时自动携带 Token
const fetchUsers = async () => {
    const res = await fetch('/api/users', {
        headers: {
            'Authorization': `Bearer ${localStorage.getItem('token')}`
        }
    });
    return res.json();
};
```

### 5. 超管查询所有租户数据

```php
use Framework\Tenant\TenantContext;

// 在控制器中
public function getAllUsers()
{
    // 临时忽略租户隔离
    $users = TenantContext::withIgnore(function () {
        return SysUser::all();
    });
    
    return response()->json($users);
}
```

## 四、安全注意事项

### 1. Workerman 环境

- ✅ 已解决：使用 Request 属性替代静态变量
- ✅ 每个请求独立，不存在跨请求污染

### 2. JWT Token 安全

- 使用强密钥（建议 256 位）
- 设置合理的过期时间
- 使用 HTTPS 传输
- 定期轮换密钥

### 3. 数据隔离

- 所有查询自动添加 `tenant_id` 条件
- 超管模式需要显式调用 `withIgnore()`
- 关联查询也需要考虑租户隔离
